# 세계사 시뮬레이터 LLM 아키텍처

## 목표
- 정적 프론트는 GitHub Pages에 유지
- 세계 이벤트, 턴 리포트, 뉴스 브리핑은 로컬 LLM으로 생성 가능하게 확장
- 모델 실행은 사용자 로컬 또는 별도 서버에서만 수행

## 분리 원칙
### 1. 프론트엔드
- GitHub Pages에서 서빙되는 정적 웹앱
- 게임 상태 관리
- 정책/외교/전쟁 규칙 처리
- LLM이 없을 때도 완전 플레이 가능

### 2. 로컬 LLM 브리지
- 사용자의 PC 또는 사설 서버에서 실행
- 프론트의 브리핑 요청을 받아 로컬 모델 API로 전달
- CORS 허용 범위를 명시적으로 관리

### 3. 모델 서버
- 예: Ollama
- 예: OpenAI-compatible local endpoint
- 예: LM Studio local server

## 요청 흐름
1. 플레이어가 턴 진행
2. 프론트는 규칙 기반 결과를 먼저 계산
3. 프론트는 선택적으로 로컬 브리지 `/api/briefing` 호출
4. 브리지는 로컬 모델 서버에 프롬프트 전달
5. 생성된 뉴스/브리핑을 프론트가 UI에 반영

## 기본 포트
- 정적 프론트: GitHub Pages 또는 로컬 파일
- 브리지 서버: `http://127.0.0.1:8787`
- Ollama 기본: `http://127.0.0.1:11434`

## 장점
- GitHub Pages는 그대로 사용 가능
- 모델은 외부에 노출되지 않음
- 로컬 모델이 꺼져 있어도 게임은 동작
- 추후 백엔드 분리나 다중 모델 지원이 쉬움

## 초기 연결 범위
- 턴 종료 후 뉴스 브리핑 1개 생성
- 실패 시 규칙 기반 브리핑으로 자동 폴백

## 후속 확장
- 지도자 성향별 내러티브
- 세력별 문체 차별화
- 전쟁 브리핑과 경제 보고서 분리
- 시드 기반 재현 가능한 서사 생성
